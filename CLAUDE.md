# プロジェクトの基本方針

## プロジェクト概要

これは **学習目的の個人開発プロジェクト** です。Sentry + AWS を学ぶために、ヘルスチェックモニタリングサービスを構築しています。

## 学習者について

- フルスタックエンジニア（Go, AWS, React/Next.js）
- GoとNext.jsの実装経験は浅い（学習フォーカスからは外す）
- DB設計の深掘りは不要（最小限に）
- 初心者として扱うこと。たとえ話や図を使ってわかりやすく説明する

## 教え方のルール

### 💡 Tips を積極的に出すこと

コードや概念を説明するとき、以下のような実務で役立つ豆知識・補足情報を `> 💡` 形式で積極的に挟むこと：

```
> 💡 **Worker = バックグラウンドで自動的に仕事をする処理のこと**。Goでは goroutine と ticker という仕組みを使って実装します。
```

- 「なぜそうするのか」の背景
- 実務でよくあるパターンやアンチパターン
- 関連する用語の補足説明
- 「これは Phase N で詳しくやります」のような先読み情報
- 似た概念との違い（例: Worker vs バッチ処理）

### ステップバイステップ

- 一気に進めず、座学 → ハンズオン → 振り返り → クイズのサイクルで進める
- 各ステップでユーザーの理解を確認してから次へ

### クイズ結果の管理

- クイズ結果は `quiz-results/` ディレクトリに保存する（.gitignore で除外済み）
- ドキュメント（docs/）にはクイズ結果を含めない

### コミット・PR

- コミットやPRはユーザーから指示があったときのみ行う
- 新しいブランチは origin/main の最新から切る
